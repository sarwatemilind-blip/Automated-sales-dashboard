# Adonis Pharma Analytics Platform — GitHub Repository Structure

## Stack
- **Frontend**: Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **Backend**: Next.js API Routes (Node.js runtime)
- **Database**: Supabase (PostgreSQL + Auth + Row Level Security)
- **Charts**: Recharts
- **File Upload**: xlsx (SheetJS) for Excel parsing
- **Export**: xlsx (Excel) + @react-pdf/renderer (PDF)
- **Hosting**: Vercel (frontend + API) + Supabase (DB + Auth)

## Database Tables (overview)

```
MASTER TABLES
├── cfas                        Distributor / CFA companies
├── stockists                   Individual stockists (linked to CFA + HQ)
├── hqs                         Headquarters (territory units)
├── employees                   All field staff (BE, ASM, RSM, ZM, VP)
├── employee_hq_map             BE → HQ assignments (up to 3 BEs per HQ)
├── hierarchy                   Manager → reportee chains
├── states                      State master
├── regions                     Region master
├── zones                       Zone master
├── products                    Canonical product list (post-clubbing)
├── product_code_map            Raw billing code → canonical product
└── brand_categories            Pillar / Others classification

TRANSACTION TABLES
├── sales_transactions          Bill-level sales (primary source, from dump uploads)
├── monthly_sales_summary       Pre-aggregated: HQ × product × month (fast reads)
└── targets                     HQ × employee × product × month (value + units)

STOCK MODULE
├── monthly_stock               One row per stockist × product × month
│                               Columns: opening_qty, primary_qty, secondary_qty,
│                                        closing_qty, sale_rate, closing_value
└── stock_upload_log            Audit trail of each dump upload

FORECASTING
├── system_forecast             3-month rolling avg (auto-calculated on upload)
└── manager_forecast            Manager override per distributor × product × month

AUTH EXTENSION
└── user_profiles               Supabase auth.users → employee_id, role, scope
```

## Key Business Rules (encoded in lib/calculations/)

| Rule | Location | Logic |
|------|----------|-------|
| BE target split | `targets.ts` | `individual_target = hq_target / no_of_bes` |
| BE achievement split | `achievement.ts` | `individual_ach = hq_ach / no_of_bes` |
| Product code clubbing | `clubProducts.ts` | Map alias codes → primary; aggregate qty + value |
| Closing qty | `stock.ts` | `closing_qty = opening_qty + primary_qty - secondary_qty` |
| Closing value | `stock.ts` | `closing_value = closing_qty × sale_rate` |
| Next month opening | `stock.ts` | Auto-populated from previous month closing_qty + closing_value |
| System forecast | `forecast.ts` | `avg(month_n-1, month_n-2, month_n-3)` per distributor × product |
| Growth % | `growth.ts` | `((cy - ly) / ly) × 100` |
| Achievement % | `achievement.ts` | `(achievement / target) × 100` |
| Gap to target | `achievement.ts` | `target - achievement` |

## Deployment Checklist

### Supabase
1. Create project at supabase.com
2. Run migrations in order: `supabase db push`
3. Run seeds: `supabase db reset --linked` (or psql)
4. Enable Email Auth in Supabase dashboard
5. Enable RLS on all tables (migrations handle this)
6. Copy URL + keys to Vercel env vars

### Vercel
1. Push repo to GitHub
2. Import project in Vercel
3. Set environment variables
4. Deploy — API routes run as Vercel Serverless Functions automatically
